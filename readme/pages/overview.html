SairFramework 框架概述

一个基于 Java 的多组件融合交互平台，以 IR 文本驱动为核心，支持热插拔插件架构。

💡 什么是 SairFramework？

SairFramework（简称 SFW）是一个使用 Java 编写的多组件融合交互平台。它的核心设计理念是避免无用冗余，通过统一的后缀名为 .ir 的文本文件驱动各个组件协同工作。

SFW 提供了一个控制台 GUI 界面，你可以在其中输入命令与各个插件交互。它支持热加载/热卸载插件，无需重启即可动态添加和移除功能。

🌟 核心特性

🔌
热插拔插件系统
插件可随时加载或卸载，无需重启框架。基于自定义 ClassLoader 实现隔离。
📜
IR 脚本驱动
使用 .ir 后缀文本文件编写脚本，支持标签跳转、变量替换、嵌入式命令执行。
🖥️
Swing GUI 控制台
内置精美的暗色主题控制台，支持彩色输出、背景图片、自定义字体和颜色。
🔄
跨平台兼容
支持 Windows、Linux 及所有 Java 支持的平台。Java 8 及以上版本均可运行。
🧰
SaComS 工具包
内置丰富的工具类库：文件管理、网络请求、数据库连接、字符串处理、图像处理、拼音转换等。
🏗️
中文代码支持
原生支持中文包名、中文类名，配合中文组件支持包实现全中文编程。

🏗️ 架构概览

📐 目录结构
SairFrameWork/
├── SFW.jar                  # 框架核心 JAR（既是运行入口也是开发依赖）
├── run.bat                   # Windows 快速启动批处理
├── data/                     # 插件数据目录（每个 Activity 自动拥有）
├── plugins/
│   ├── bootlib/              # 系统级依赖（使用系统 ClassLoader 加载）
│   ├── modlib/               # 普通依赖库（使用 SairLoader 加载）
│   └── exection/             # 插件本体目录（存放开发的插件 JAR）
├── source_code/
│   ├── source_code_SFW/      # 框架核心源码
│   ├── source_code_SaComS/   # SaComS 工具包源码
│   └── source_code_execs/    # 各种插件源码示例
└── readme/                   # 本文档目录

⚙️ 工作原理

🔄 启动流程
  1. JVM 检测 — 检测当前 Java 版本。非 Java 8 时会自动重启并使用 -Xbootclasspath/a 加载 bootlib
  2. 加载 bootlib — 使用系统 ClassLoader 加载 plugins/bootlib/ 中的 JAR
  3. 加载 modlib — 使用 SairLoader（自定义 ClassLoader）加载 plugins/modlib/ 中的 JAR
  4. 加载 exection — 扫描 plugins/exection/ 中的 JAR，读取 MANIFEST.MF 中的 ACT 属性，实例化 Activity
  5. 启动控制台 — 显示 GUI 控制台窗口，等待用户输入命令
  6. 自动执行 — 如通过双击 .ir 文件启动，则自动执行脚本内容
📋 命令处理流程
用户输入命令 → SystemSpliter 解析 → 获取 Activity名称/功能名/参数
                    ↓
           Libraries.activities 查找对应 Activity
                    ↓
           OderFact.runner() 处理内置命令（help/exit/info/ofunc...）
                    ↓
           Activity.main(funcName, args) 调用插件自定义功能
🧩 类加载器层次
加载器类型加载目录用途
System ClassLoader 系统 plugins/bootlib/ JVM 功能修补、SPI、线程上下文操作
SairLoader 自定义 plugins/modlib/ 普通依赖库，支持热卸载
ExectionLoader 自定义 plugins/exection/ 插件隔离加载，每个插件独立 ClassLoader
