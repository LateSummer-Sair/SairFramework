SaComS 工具包

SairFramework 内置的综合性 Java 工具类库，提供文件管理、数据结构、字符串处理、网络、数据库、图像等全方位能力。

📦 引入方式

SaComS.jar 位于 plugins/modlib/SaComS.jar，由框架自动加载。插件中可直接 import sair.sacoms.* 使用。

📑 工具类目录

🔧 其他实用工具

📻 FTP — FTP 客户端 sair.sacoms.FTP

基于 Apache Commons Net 的 FTP 客户端封装。提供连接认证、文件上传（支持本地文件路径和 InputStream 两种方式）、文件下载、切换工作目录、递归创建多级目录、删除远程文件等完整功能。默认使用匿名登录（anonymous），可自定义主机地址、端口（默认21）、用户名、密码和字符编码（默认GBK）。

📖 CodeReader — 文本编码检测器 sair.sacoms.CodeReader

文本文件编码自动识别工具。构造时传入候选编码列表（如 "GBK"、"UTF-8" 等），调用 detectCharsetForFile(path) 逐个尝试用候选编码解码文件内容，返回第一个成功解码的编码名称。也支持通过 detectCharsetForStream(InputStream) 直接对输入流进行检测。底层原理：遍历读取字节缓冲区，用 CharsetDecoder 解码，若抛出 CharacterCodingException 则说明编码不匹配。

📋 Tables — 系统剪贴板工具 sair.sacoms.Tables

系统剪贴板读写工具（并非"表格数据"）。基于 java.awt.Toolkit 和 Transferable 接口，提供四个静态方法：

  • getSysClipboardText() — 读取剪贴板中的文本内容
  • setSysClipboardText(String) — 将字符串写入剪贴板
  • getImageFromClipboard() — 从剪贴板获取图片（Image 类型）
  • setClipboardImage(Image) — 将图片写入剪贴板

🎵 Aplayer — MP3 音频播放器 sair.sacoms.Aplayer

基于 JLayer（javazoom.jl.player.Player）的标准 MP3 文件播放器，不是 JavaFX。实现了 Runnable 接口，可以在新线程中播放 MP3 文件。支持多种初始化方式：文件路径字符串、File 对象、Urler 对象或 InputStream 流。提供 toStop() 停止播放、isEnd() 判断播放是否已结束。

🎤 KRC — 酷狗歌词解密解析 sair.sacoms.KRC

酷狗音乐 KRC 逐字歌词文件的解密与解析工具。KRC 文件经过 XOR 异或加密 + zlib 压缩处理，该工具内部流程为：① 用 SFS.miarry 密钥数组逐字节异或解密；② 使用 Inflater 解压缩；③ 将解密后的文本按行解析，分离出元信息头（getInfoTi()）和逐字歌词正文（getLrcBody()）。还提供 getHead_Time() 提取一行中的时间标记，get_Chars_Time() 提取逐字时间与字符的对应关系。

🎶 LRC — 标准歌词解析 sair.sacoms.LRC

标准 LRC 歌词文件解析器。读取 LRC 文件后自动提取元数据：title（歌曲标题，ti标签）、artist（艺术家，ar标签）、album（专辑，al标签）、lrcMaker（制作者，by标签）。同时解析每行中的时间标签（格式 [mm:ss.xx]）与歌词文本的对应关系，输出 List<LrcLine> 并按时间升序排列。

🔀 Seq — 快速排序器 sair.sacoms.Seq

基于快速排序算法的排序工具（并非"序列生成"）。支持对 Integer[] 数组、实现了 SeqPageI 接口的对象数组、以及 SairLists<T> 列表进行原地排序。通过 Key 机制选择排序方向：

  • Seq.BySort — 升序排列（从小到大）
  • Seq.ByOrder — 降序排列（从大到小）

自定义对象需实现 SeqPageI 接口的 getSeqID() 方法，返回排序依据的长整型 ID 值。排序完成后自动调用 System.gc() 回收临时内存。

// 排序示例
Integer[] arr = {3, 1, 4, 2};
Seq.Of(Seq.BySort, arr);  // → [1, 2, 3, 4]
Seq.Of(Seq.ByOrder, arr); // → [4, 3, 2, 1]

💾 Objserialize — 对象序列化工具 sair.sacoms.Objserialize

基于 JDK 原生序列化（ObjectOutputStream / ObjectInputStream）的对象持久化工具。使用方法：

  • toSerial(T obj, String url) — 将实现了 Serializable 接口的对象序列化到指定路径；若目录不存在则自动创建
  • unSerial(String url) — 从文件中反序列化还原对象，返回泛型类型，使用时需强制类型转换；若文件不存在返回 null

🀄 PY — 汉语拼音转换工具 sair.sacoms.PY

汉语拼音与汉字互转工具（PY = 拼音，并非 Python）。内置拼音-汉字对照字典（数据来源 /sair/res/PY_CH.txt），提供以下能力：

  • PYgetCH(String py) — 根据拼音返回该读音对应的所有汉字（char[]）
  • CHgetPY(Character ch) — 根据汉字返回其所有可能的拼音（String[]），含多音字支持
  • getPtS() / getStP() — 获取完整的拼音→汉字 / 汉字→拼音映射表

支持通过构造参数传入自定义码表，覆盖默认字典。

🔐 Key<T> — 类型安全键 sair.sacoms.Key<T>

泛型化的类型安全键值封装。在 SaComS 中被广泛用于替代字符串常量进行模式选择（如 Seq 中的 BySort/ByOrder、CMD 中的各种命令模式等）。

  • Key.creatKey(T abs) — 创建绑定具体值的键
  • Key.creatKey() — 创建空键（值为 null）
  • about() — 获取键绑定的值

因为每个 Key 实例是唯一的对象引用，使用 equals() 比较时比字符串更安全，避免了魔法字符串带来的拼写错误风险。

// Key 的基本用法
Key<String> myKey = Key.creatKey("value");
String value = myKey.about();  // → "value"

🔍 匹配器系列 — 括号匹配与代码行分析

SairBracketMatcher — 括号匹配分割器 sair.sacoms.SairBracketMatcher

根据自定义括号规则对字符串进行括号配对验证与层级分割。构造时指定括号规则（如 {"()", "[]", "{}"}）和转义字符（capChar，如 '%'）。核心方法：

  • chkSplit() — 验证所有括号是否完全匹配配对
  • split() — 按括号层级将内容分割为字符串数组，从最内层括号内容到最外层

支持运行时通过 reSetString() / reSetRule() 动态切换匹配目标和规则。

SairLineMatcher — 代码行结构分析器 sair.sacoms.SairLineMatcher

基于 SairBracketMatcher 的分割结果，进一步对单个代码行进行语义分析。核心方法 getSuperFuncName(index, escapeChar, outRule)：提取括号前的方法名或变量名（例如分析 foo(arg1, arg2) 时，返回 "foo"）。

SairMatcherStatic — 内部辅助类 sair.sacoms.SairMatcherStatic

包级私有工具类，为上述匹配器提供静态帮助方法：字符在括号规则数组中的查找（chk()）、CharPage 内部类用于记录每个括号对的起止位置。

📦 Txtkey — 文本文件序列化打包 sair.sacoms.Txtkey

将文本文件内容读入内存并通过 Objserialize 序列化为二进制文件的工具（并非"键值对解析"）。两个核心静态方法：

  • fxTxt(String inUrl, String outUrl, Key<String> enc) — 将指定编码的文本文件读取为字符串数组，然后序列化保存为二进制文件
  • reTxt(String inUrl, String outUrl) — 反向操作，将序列化后的二进制文件还原为原始文本文件

这本质上是一个文本→序列化格式的打包/解包工具，可用于文本数据的加密存储或减少体积。

🔢 StringNumber — 字符串字符统计 sair.sacoms.StringNumber

接收一个 Collection<String>，自动去除所有标点符号和空白字符后统计剩余有效字符的总长度。核心方法：

  • getLen() — 返回所有字符串去除标点和空白后的总字符数
  • getStrs() — 返回经过清理后的字符串列表

适合用于统计文本的有效内容长度、过滤噪声等场景。

✂️ SplitsJMD — IR 命令语法解析器 sair.sacoms.SplitsJMD

SairFramework 核心命令解析器，将类似 "/命令名 参数1 参数2 ..." 格式的 IR 命令字符串拆解为三个组成部分：

  • getHead() — 命令头（第一个 / 之前的内容，通常为调用者标识）
  • getEnd() — 命令名称（/ 之后、空格之前的部分）
  • getOtherRuns() — 参数数组（空格分隔的后续参数）

实现了 ObjectBySplits 接口。还提供 ReturnOtherRunsToJMD() 将参数数组反向拼接为命令字符串（可跳过指定下标），以及 isNullEndAndNullArgs() 判断空命令名+空参数的情况。

// 解析示例："Caller /doSomething arg1 arg2"
SplitsJMD spl = SplitsJMD.split("Caller /doSomething arg1 arg2");
spl.getHead();       // → "Caller"
spl.getEnd();        // → "doSomething"
spl.getOtherRuns();  // → ["arg1", "arg2"]
