快速开始

从零开始搭建开发环境，创建你的第一个 SairFramework 插件。

📋 前置要求

  • JDK 8 或更高版本（推荐 JDK 8，热卸载支持最好）
  • 任意 Java IDE（Eclipse、IntelliJ IDEA 等）
  • 下载 SairFramework 完整目录（包含 SFW.jar 和 plugins/ 目录）

🚀 第一步：创建插件项目

📁 项目结构

创建一个标准 Java 项目，以下面的结构为例：

MyPlugin/                          # 你的插件项目根目录
├── src/
│   └── sair/
│       └── myplugin/
│           └── MyActivity.java      # 插件主类
└── META-INF/
    └── MANIFEST.MF              # 插件描述文件（必须！）

✏️ 第二步：编写 MANIFEST.MF

📝 MANIFEST.MF 文件

在 META-INF/MANIFEST.MF 中声明插件的入口 Activity 类：

Manifest-Version: 1.0
ACT: sair.myplugin.MyActivity;
💡 多个 Activity

一个 JAR 可以包含多个 Activity，用分号分隔：

Manifest-Version: 1.0
ACT: sair.myplugin.MainActivity;sair.myplugin.SubActivity;
⚠️ 注意

ACT 的值末尾必须有分号（;）！这是框架解析的标识。

💻 第三步：编写 Activity 类

📄 MyActivity.java

所有插件的主类必须继承抽象类 sair.user.Activity：

package sair.myplugin;

import sair.user.Activity;
import sair.sys.SairCons;
import sair.FCM;
import sair.Pathes;

public class MyActivity extends Activity {

    // ① 核心方法：处理用户命令
    @Override
    public Object main(String funcName, String args) {
        switch (funcName) {
            case "hello":
                SairCons.println("Hello, SairFramework!");
                return true;
            case "greet":
                SairCons.println("你好, " + args + "!");
                return true;
        }
        return false; // 返回 false 会自动打印 help 信息
    }

    // ② 帮助信息
    @Override
    public String[] help() {
        String name = this.getName();
        return new String[] {
            Pathes.printSplit,
            "MyPlugin v1.0 - 我的第一个插件",
            Pathes.printSplit,
            name + "/hello : 输出 Hello World",
            name + "/greet [名字] : 向指定名字打招呼",
        };
    }

    // ③ 退出时清理资源
    @Override
    public void exit() {
        SairCons.println("MyPlugin 已退出");
    }

    // ④ 可选：指定数据目录（不重写则默认为类全名）
    @Override
    protected String dataDir() {
        return "myplugin"; // 数据存放在 data/myplugin/
    }
}

📦 第四步：打包与部署

🏗️ 打包要求
  1. 导出普通 JAR — 在 IDE 中将项目导出为普通 JAR 文件（不是可执行 JAR）
  2. 仅打包自己的代码 — 不需要将 SFW.jar 或 SaComS.jar 一起打包
  3. 确认 MANIFEST.MF — 打包后检查 JAR 内的 META-INF/MANIFEST.MF 是否包含 ACT 声明
  4. 放入 plugins/exection/ — 将生成的 JAR 文件放入 plugins/exection/ 目录

▶️ 第五步：运行测试

🖥️ 运行 SFW

打开命令行，在 SairFramework 目录下执行：

java -jar SFW.jar

或者直接双击 run.bat（Windows）。

启动后在控制台输入以下命令测试：

# 查看已加载的插件（注意插件名是 JAR 文件名）
/list

# 假设 JAR 文件名为 myplugin.jar
myplugin/hello
myplugin/greet 世界

# 查看插件帮助
myplugin/help

# 查看插件信息
myplugin/info

🔧 调试技巧

🐛 在 IDE 中调试插件

在插件类中添加 main 方法进行调试，无需启动完整 SFW：

public static void main(String[] args) throws Exception {
    // 创建自己的 Activity 实例
    Activity myacti = new MyActivity();

    // 方式一：使用 Main.toTest 进行指令测试
    Main.toTest(myacti, "hello", "");
    Main.toTest(myacti, "greet", "Sair");
}
💡 方式二：注册到 Libraries

更完整的调试方式，可以测试多组件协同：

public static void main(String[] args) throws Exception {
    Activity myacti = new MyActivity();
    myacti.setName("test");
    Libraries.activities.put("test", myacti);

    // 加载依赖组件
    Main.toTest(myacti, "", "");

    // 使用 runner 调用（可调用其他组件）
    SairCons.runner(false, "test/hello");
    SairCons.runner(false, "test/greet Sair");
}
