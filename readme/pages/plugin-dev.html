插件开发指南

深入理解 SairFramework 插件开发的核心概念与最佳实践。

📦 Activity 抽象类

🧬 Activity 是插件的核心

每个插件至少包含一个继承自 sair.user.Activity 的类。Activity 提供了插件所需的基础设施：

方法说明是否必须重写
main(String funcName, String args)命令处理入口，funcName 为功能名，args 为参数必须
help()返回帮助信息字符串数组必须
exit()插件被卸载时调用，用于清理资源必须
dataDir()自定义数据目录，默认为类的全限定名可选
o_funcMain(Object o)处理 ofunc 内置命令的返回值可选
🔄 main() 方法返回值规则
返回值框架行为
true 或非 Boolean 对象正常执行完毕，无额外输出
false功能未找到或执行失败，自动打印 help 信息
null无任何效果（静默）
🗂️ 数据目录

每个 Activity 自动拥有独立的数据目录，位于 data/ 下。通过 getDataDir() 获取。目录在首次访问时自动创建。

// 默认数据目录: data/sair.myplugin.MyActivity/
String path = this.getDataDir();

// 通过重写 dataDir() 自定义:
@Override
protected String dataDir() {
    return "myplugin"; // → data/myplugin/
}

📋 命令系统详解

⌨️ 命令格式

SFW 命令格式为：插件名/功能名 参数

# 完整命令格式
myplugin/hello
myplugin/greet World

# 以 / 开头调用 SFW 内置命令（不指定插件名）
/exit          # 退出框架
/list          # 列出已加载插件
/print Hello   # 在控制台打印
/clear         # 清屏
🔄 命令解析流程

SFW 使用 SystemSpliter 解析命令字符串，分解为三部分：

部分来源示例
getExecName()/ 之前的内容myplugin
getExecFunc()/ 之后、空格之前hello
getArgs()空格之后的全部内容World
🏷️ 内置命令

每个 Activity 默认拥有以下内置命令（由 OderFact 处理，不需要在 main() 中处理）：

命令说明
help显示插件帮助信息
exit调用插件的 exit() 方法
info显示插件的 data 目录和 JAR 路径
close临时关闭插件的命令接收（插件仍然在内存中）
open重新打开插件的命令接收
oset 新名 原名为已有命令创建别名
orem 别名删除命令别名
ofunc '命令'执行一个命令并将其返回值传入 o_funcMain()

🔄 插件生命周期

⏳ 生命周期阶段
  1. 发现 — SFW 扫描 plugins/exection/ 中的 JAR 文件
  2. 读取 — 解析 MANIFEST.MF 中的 ACT 属性
  3. 加载 — ExectionLoader 加载 JAR，使用 LoaderManager.loadMain() 实例化 Activity
  4. 命名 — Libraries.setActivityName() 自动分配名称（默认为 JAR 文件名，重名则追加数字）
  5. 注册 — 将 Activity 存入 Libraries.activities 和 Libraries.exections
  6. 运行 — Activity 就绪，可接收命令
  7. 卸载 — 调用 exit() 后从 Libraries 中移除，关闭 ClassLoader
🏷️ 插件命名规则

插件名称默认为 JAR 文件名（去掉 .jar 后缀）。如果多个插件同名，框架会自动追加数字后缀（-1, 0, 1, 2...）。

可以使用内置命令 /rename 旧名 新名 临时重命名，但重启 SFW 后失效。

✨ 最佳实践

🎯 开发建议
  • 分离逻辑 — 将 main() 中的 switch 分支委托给独立的 Action 类处理（参考 ActivityAction 模式）
  • 参数解析 — 使用 ToolPack.pathRepack() 处理带双引号的文件路径
  • 使用 SaComS — 善用 SaComS 工具包提供的文件操作、字符串处理等能力
  • 资源清理 — 在 exit() 中关闭所有打开的资源（文件句柄、网络连接等）
  • 色彩规范 — 使用 FCM 类中定义的色彩常量保持界面一致性
  • 依赖放置 — 如果插件有额外依赖 JAR，放入 plugins/modlib/（普通依赖）或 plugins/bootlib/（系统级依赖）
⚠️ 注意事项
  • 插件打包时不要包含 SFW.jar 或 SaComS.jar——这些由框架提供
  • MANIFEST.MF 的 ACT 值末尾必须有分号
  • 多个 Activity 的类名用分号分隔
  • 插件名称由 JAR 文件名决定，避免使用特殊字符
