📜 IR 脚本语言

📝 什么是 IR 脚本？

IR 脚本是 SairFramework 的自动化脚本语言，文件后缀为 .ir。它是一个纯文本文件，每一行都是一条 SFW 命令。SFW 会按行顺序执行这些命令。

IR 脚本可以直接用记事本打开编辑。你可以通过双击 .ir 文件（关联到 run.bat）来启动 SFW 并自动执行脚本。

📄 基本语法
// 这是注释：行首以双斜杠开头即为注释，不会被执行

// 空行会被忽略

// 以下都是有效的 IR 命令
/print-c 0 255 0 欢迎使用 SairFramework!
/sleep 1000
/println 正在初始化插件...
myplugin/hello
myplugin/greet IR脚本用户

// 变量定义与使用
/var-add myPath D:\Test
/println %myPath%
🏷️ 标签（Label）与跳转

IR 脚本支持标签定义与跳转，实现控制流。使用 {} （花括号）定义标签块，使用 /TO:标签名 跳转。

// 定义一个名为 "loop" 的标签块
loop {
    /println 循环中...
    /sleep 500
    /println 再次循环!
    /TO:loop          // 跳回 loop 标签继续执行（无限循环）
}

// 主流程中调用标签
/println 开始执行
/TO:loop             // 跳转到 loop 标签块
/println 这行不会执行（因为在无限循环中）
⚠️ 标签语法要点
  • 标签名后紧跟 { 开始标签块（花括号前可以有空格）
  • } 单独一行结束标签块
  • 使用 /TO:标签名 跳转到指定标签
  • 支持嵌套标签
🔄 变量替换

使用 /var-add 变量名 值 定义变量，在命令中用 %变量名% 引用。

// 定义变量
/var-add name Sair
/var-add version 5.3

// 使用变量
/println 作者: %name%
/println 版本: %version%

// 变量在路径中的应用
/var-add workDir D:\WorkSpace
/print %workDir%\project
📌 嵌入式命令

使用 '命令'（单引号包裹）可以在命令参数中嵌入另一个命令，嵌入命令的结果会被自动替换。

// 嵌入式命令：将内部命令的结果作为外部命令的参数
/println 'pl/getNowPlayID'

// 等价于:
// 1. 先执行 pl/getNowPlayID，获得返回值（如 "12345"）
// 2. 再执行 /println 12345
🎛️ IR 脚本管理命令
命令说明
/ir "路径"在新线程中执行 IR 脚本文件
/ir-i "路径"使用当前线程执行 IR 脚本（会阻塞当前线程）
/ir-x 脚本名停止指定的 IR 脚本
💡 自动执行

SFW 启动时如果没有传入参数，会自动在 data/framework/ 下查找并执行 autorun.ir 文件。你可以在此文件中编写启动时需要自动执行的任务。

🔌 SpliterSPI — 自定义命令解析器

🔧 扩展命令解析

SFW 默认使用 SystemSpliter 解析 插件名/功能名 参数 格式的命令。你可以通过实现 SpliterSPI 抽象类来自定义命令解析方式。

实现步骤

  1. 创建一个类继承 sair.user.SpliterSPI
  2. 实现 getSpliter(String cmd) 返回自定义的 Spliter 实现
  3. 实现 getUninstallCMD() 返回卸载此解析器的命令
  4. 将编译后的类放入 plugins/modlib/
  5. 在 SFW 控制台输入 /setspliter 类全限定名 安装自定义解析器
package mypackage;

import sair.user.SpliterSPI;
import sair.sys.tools.Spliter;

public class MySpliterSPI extends SpliterSPI {

    @Override
    public Spliter getSpliter(String cmd) {
        // 返回自定义的 Spliter 实现
        return new MySpliter(cmd);
    }

    @Override
    public String getUninstallCMD() {
        return "/my-spliter-uninstall";
    }
}

📂 插件目录详解

📁 三个 plugins 子目录的用途

1. plugins/bootlib/ — 系统级依赖

此目录中的 JAR 使用系统 ClassLoader 加载。适合放入需要对 JVM 功能进行修补、需要线程上下文操作或 SPI 机制的库。

⚠️ 注意

Java 8 以上版本中，bootlib 中的 JAR 无法被热卸载（需要重启 JVM）。Java 8 可以热卸载。

2. plugins/modlib/ — 普通依赖库

此目录中的 JAR 使用 SairLoader（自定义 ClassLoader）加载。适合放入普通的第三方依赖库，如 SaComS.jar、gson 等。

可以随时通过 SairLoader.removeJarFiles() 热卸载。

3. plugins/exection/ — 插件本体

此目录存放你开发的插件 JAR。每个插件使用独立的 ExectionLoader 加载，实现插件间类隔离。

插件必须包含带有 ACT 声明的 MANIFEST.MF 文件。

🔄 Mod 依赖库的特殊机制

Mod 类型的 JAR（放入 modlib 的库）如果在其 MANIFEST.MF 中声明了 NAME 和 DIR 属性，可以被其他插件通过 SairLoader.getMOD_MEAT_INFFILE(name) 获取其内部资源路径。

// MANIFEST.MF 示例
Manifest-Version: 1.0
NAME: MyLib;
DIR: sair/res/;

// 在其他插件中获取
String dir = SairLoader.getMOD_MEAT_INFFILE("MyLib"); // → "sair/res/"

🚀 框架启动方式

▶️ 三种启动方式
方式说明
java -jar SFW.jar 标准启动，打开控制台，自动执行 autorun.ir
java -jar SFW.jar "路径.ir" 启动并自动执行指定的 IR 脚本文件
双击 .ir 文件 需先将 .ir 文件关联到 run.bat，双击即可启动并执行
💡 Linux 用户

在 Linux 上需要手动将 .ir 文件路径传给 SFW.jar：java -jar SFW.jar /path/to/script.ir

📡 打印代理（PrintRunnable）

🔀 拦截控制台输出

实现 PrintRunnable 接口可以拦截所有控制台输出，用于日志记录、远程转发等场景。

public class MyLogger implements PrintRunnable {
    @Override
    public void run(Integer index, Color c, String info) {
        // 记录所有控制台输出
        System.out.println("[LOG]" + info);
    }
}

// 注册打印代理
SairCons.addPrintRunnable("myLogger", new MyLogger());

// 也可以通过 SFW 命令注册
// /print-cpr className

🏗️ 中文编程支持

🀄 中文包名与类名

SairFramework 原生支持中文包名和类名。例如：

package 夏希尔.中文测试项目;

import sair.user.Activity;

public class 测试组件主函数 extends Activity {
    // ...
}

配合 plugins/modlib/中文代码组件支持包.jar 可以实现全中文编程。
